The file `is_backend.hpp` is part of Boost.Multiprecision and not of Boost.Geometry  
  
So you can open an issue there (AFAIK you cannot move issues from sublib to sublib).  
https://github.com/boostorg/multiprecision/issues  
  
Boost.Geometry has currently a hard dependency on Boost.Multiprecision (I think we should change that, @vissarion ? It should be somehow only be triggered if Boost.Multiprecision is to be used. It's now even in `boost/geometry/core/coordinate_promotion.hpp`, I was not aware of that or had forgotten it)
