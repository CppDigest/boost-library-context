## Comment 3

> Username: mloskot  
> Created at: 2022-10-20 20:20:49 UTC

The Boost CMake configuration is a secondary build configuration, experimental, with those CML files generated by scripts, with dependencies in those CML determined automatically via scanning `#include`-s what obviously is not clever enough to capture optional modes of complex inter-libraries dependencies.  
  
Simply, do not use the Boost CMake. Instead, use the traditional Find-module [FindBoost.cmake](https://github.com/Kitware/CMake/blob/468a995346271c02f2248adf3539fbb34edaef2e/Modules/FindBoost.cmake) which explicitly defines and hard-codes dependencies between the Boost libraries, and which, as you can notice, does not declare any for Boost.Geometry.  
  
Check https://cmake.org/cmake/help/latest/module/FindBoost.html#boost-cmake for `Boost_NO_BOOST_CMAKE`.  
  
So, I'd argue the current docs are accurate and complete. Unless, a chapter about using Boost.Geometry with Boost.CMake has been added and I have missed that fact.
