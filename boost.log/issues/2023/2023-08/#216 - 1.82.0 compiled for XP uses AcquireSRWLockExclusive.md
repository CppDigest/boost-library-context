I compiled boost 1.82.0 for WinXP with MSVC 2019 using the folowing script. That worked for 1.78.0 but 1.82.0 does not run on WinXP as AcquireSRWLockExclusive is not found (just Win Vista or newer available).  
  
```  
SET ARCH=architecture=x86  
SET INST=instruction-set=core2   
  
SET JOBS=-j16  
  
.\b2 headers  
  
rmdir /S /Q bin.v2 stage  
  
b2 define=BOOST_USE_WINAPI_VERSION=BOOST_WINAPI_VERSION_WINXP %ARCH% %INST% %JOBS%  --toolset=msvc-14.2 address-model=32 --build-type=complete --without-iostreams --without-mpi --without-wave --without-math --without-python debug-symbols=on stage  
  
rmdir /S /Q bin.v2  
  
b2 define=BOOST_USE_WINAPI_VERSION=BOOST_WINAPI_VERSION_WINXP %ARCH% %INST% %JOBS% --toolset=msvc-14.2 address-model=64 --build-type=complete --without-iostreams --without-mpi --without-wave --without-math --without-python debug-symbols=on stage  
  
rmdir /S /Q bin.v2  
```

Just so you know, you shouldn't use symbolic names for macro values in command line switches such as `define=BOOST_USE_WINAPI_VERSION=BOOST_WINAPI_VERSION_WINXP`. Since `BOOST_WINAPI_VERSION_WINXP` is not defined in the command line, `BOOST_USE_WINAPI_VERSION` is defined to a literal token `BOOST_WINAPI_VERSION_WINXP` instead of the integer version. Use `define=BOOST_USE_WINAPI_VERSION=0x0501` instead.
