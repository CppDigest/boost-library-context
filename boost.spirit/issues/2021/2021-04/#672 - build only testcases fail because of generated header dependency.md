# #672 - build only testcases fail because of generated header dependency [Closed]

> Created at: 2021-04-30 23:27:35 UTC

The regression_matlib_static and regression_matlib_switch testcases are dependent on header files generated by the regression_matlib_generate and regression_matlib_generate_switch testscases respectively. This leads to a compile error when building the tests without running:  
```  
cd libs/spirit  
../../b2 test -j12 -a -d1 testing.execute=off  
```  
Where the error will be something like:  
`../libs/spirit/test/lex/regression_matlib_static.cpp:15:10: fatal error: 'matlib_static.h' file not found`  
  
https://github.com/boostorg/spirit/blob/e30073e5bebb8102c73d1ad3588bf3f68cec2b64/test/lex/Jamfile#L90  
  
This issue was found when running these tests for VxWorks, since tests are built on a host machine and then executables are transferred to VxWorks to test.
