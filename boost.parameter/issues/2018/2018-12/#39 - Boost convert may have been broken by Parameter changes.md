I see that you have been back and forth about this with a PR for Boost convert. Sorry for bringing this up, but we need to resolve this issue somehow. Can you discuss this on the Boost mailing list so that others can be involved. I do not want you to have to back out any of your changes to 'develop' but it is also important that others know that one of your decisions has changed something for another Boost library, and what are the ramifications for that change. I have approved your changes because I think your work to improve Parameter is excellent, but when other Boost libraries are affected, even other Boost libraries which might have depended on details of Boost Parameter which they should normally not have done, it needs to be discussed on the mailing list.
