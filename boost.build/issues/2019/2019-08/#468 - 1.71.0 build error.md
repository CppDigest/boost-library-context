## Comment 4

> Username: Lastique  
> Created at: 2019-08-30 14:33:45 UTC  
> Url: https://github.com/boostorg/build/issues/468#issuecomment-526624901  

Apparently, the problem is that `string_*` declarations are declared as `extern "C"` and therefore are mangled according to C rules, not C++. I checked this by separately compiling every .cpp file. This happens because `strings.h` file where these functions are declared, clashes names with the system header `strings.h`, which gets indirectly included by other system headers within an `extern "C"` region. For example, `/usr/include/string.h` can include `<strings.h>` under some conditions.  
  
I'm not sure why this doesn't trigger reliably in CI or in my git checkout. Possibly, include search order is not fully deterministic in gcc, if the same-named headers are found in multiple locations.
