Hi! I am trying to build boost with the new Intel compiler(icpx) and I am encountering an issue. Here are the steps I follow:  
  
1. Download and unzip Boost 1.79.0.  
2. `./bootstrap.sh --with-libraries=context,filesystem --with-toolset=intel-linux`  
3. `./b2 link=static toolset=intel-linux --prefix=/path/to/install --user-config=/path/to/user-config.jam install`  
  
After running the 3rd step I get the following error:  
  
```  
error: at project-config.jam:12  
error: duplicate initialization of intel-linux with the following parameters:  
error: version = 2022.0.0  
error: previous initialization at /mnt/andrei/builds/boost/foundationdb/build_output/user-config.jam:1  
```  
  
user-config.jam looks like this :   
`using intel-linux : : /opt/intel/oneapi/compiler/2022.0.2/linux/bin/icpx : <cxxflags>-fvisibility=hidden <cxxflags>-fPIC <cxxflags>-std=c++17 <cxxflags>-w <cxxflags>-stdlib=libc++ <cxxflags>-nostdlib++  ;`  
  
  
project-config.jam looks like this:  
  
```  
# B2 Configuration  
# Automatically generated by bootstrap.sh  
  
import option ;  
import feature ;  
  
# Compiler configuration. This definition will be used unless  
# you already have defined some toolsets in your user-config.jam  
# file.  
if ! intel-linux in [ feature.values <toolset> ]  
{  
    using intel-linux ;  
}  
  
project : default-build <toolset>intel-linux ;  
  
// some more options settings...  
```  
  
Removing the if-statement in the project-config.jam file will solve the problem, but I don't know if this would affect my build. When building with clang, the issue is not reproduced anymore even if the `using clang` statement is present in both user-config.jam and project-config.jam. Do you have any idea what I am doing wrong?
