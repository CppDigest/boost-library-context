Right now, only narrow `String`s are supported. It might be nice to support fancier types of strings, like wide or UTF-XX strings. If we support this, the different ways of creating `String`s should also support these fancier char types (like the `_s` udl).  
  
I'm not sure whether this is useful, so please comment on this issue if you need this. If enough people want it, I might implement that feature.
