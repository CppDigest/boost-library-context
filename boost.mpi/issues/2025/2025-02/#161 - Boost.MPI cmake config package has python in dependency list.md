I could not find where the configuration of the CMake dependencies stands, whether it's here or in the Boost.MPI repo, so I might be reporting this in the wrong place...  
  
I am building Boost.MPI without Python binding  for Conda in https://github.com/conda-forge/libboost-mpi-feedstock.  
  
The issue is in the gernerated config file package `lib/cmake/boost_mpi-1.87.0/libboost_mpi-variant-shared-py3.12.cmake` that has `python` in the list of Boost:mpi target dependency:  
  
```  
list(APPEND _BOOST_MPI_DEPS container graph python serialization headers)  
                                            ^   
```  
  
`Boost::mpi` does not depend on Boost.Python, only the python binding (module) does.
