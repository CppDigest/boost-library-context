## Comment 5

> Username: oxygene  
> Created_at: 2021-01-26 06:03:14 UTC  
> Url: https://github.com/boostorg/property_tree/pull/50#issuecomment-767317099  

> What is the most acceptable way to suppress the warnings for now?  
  
This is how I fixed it for me:  
```cpp  
#define BOOST_BIND_GLOBAL_PLACEHOLDERS // can be removed once this is fixed: https://github.com/boostorg/property_tree/pull/50  
#include <boost/property_tree/json_parser.hpp>  
#undef BOOST_BIND_GLOBAL_PLACEHOLDERS  
```  
If you modify your central header file like this, there is no need to set `BOOST_BIND_GLOBAL_PLACEHOLDERS` globally. Instead, narrow down the scope of the "warning exception" as far as possible. This is as narrow as it gets. Ugly, but there's not much we can do besides hoping that deprecations like this receive better testing before the next boost release.
