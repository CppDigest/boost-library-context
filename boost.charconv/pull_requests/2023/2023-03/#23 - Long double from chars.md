## Comment 16

> Username: pdimov  
> Created_at: 2023-05-11 15:58:52 UTC  
> Url: https://github.com/boostorg/charconv/pull/23#issuecomment-1544257998  

I'll take a closer look later, no time at the moment, but what I think you want here is to make the _library build_ link to `quadmath` and not the test executables. Users don't do anything with the test executables, so if the library is not built with float128 support, it won't support float128 regardless of what you put into `test/Jamfile`.  
  
How the library is built is described in `build/Jamfile`, not in `test/Jamfile`.  
  
In addition, you should never use `<linkflags>` to link to a library. This won't work correctly for all shared/static cases. You want  
  
```  
lib quadmath ;  
```  
  
and then instead of `<linkflags>-lquadmath`, `<library>quadmath`.
