1. Where does the repetition come from?  
  
By commenting  
`if (capacity[*ei] > 0)`,   
the additional outputs (20 lines) with capacity[*ei] == 0 are shown, which are invalid results and are originally skipped.  
  
Referring to the following codes  
```  
for (boost::tie(u_iter, u_end) = vertices(g); u_iter != u_end; ++u_iter)  
  for (boost::tie(ei, e_end) = out_edges(*u_iter, g); ei != e_end; ++ei)  
    if (capacity[*ei] > 0)  
      std::cout << "f " << *u_iter << " " << target(*ei, g) << " "  
        << (capacity[*ei] - residual_capacity[*ei]) << std::endl;   
```  
if capacity[*ei] == 0, the value, `(capacity[*ei] - residual_capacity[*ei])` is invalid, because the edge does not have the capacity to get the flow. These edges with capacity[*ei] == 0 are reverse edges.  
  
To see where the edges with capacity[*ei] == 0 come from, we can see how the program read the input data.  
  
In the file, max_flow.dat, the content is as follows:  
```  
c This file was generated by genrmf.  
c The parameters are: a: 2 b: 2 c1: 1 c2: 5  
p max       8         20  
n       1 s  
n       8 t  
a       1       7          3  
a       1       2         20  
a       1       3         20  
a       2       6          1  
a       2       1         20  
a       2       4         20  
a       3       5          4  
a       3       4         20  
a       3       1         20  
a       4       8          5  
a       4       3         20  
a       4       2         20  
a       5       6         20  
a       5       7         20  
a       6       5         20  
a       6       8         20  
a       7       8         20  
a       7       5         20  
a       8       7         20  
a       8       6         20  
```  
  
The file, max_flow.dat, is read by the following function in boykov_kolmogorov-eg.cpp  
`  
read_dimacs_max_flow(g, capacity, rev, s, t); // row 92  
`  
  
This function calls read_dimacs_max_flow_internal(...) in read_dimacs.hpp, and reads the .dat file as follows:  
```  
int read_dimacs_max_flow_internal(...) {  
....  
while (std::getline(in, in_line)) {  
    ++no_lines;  
    switch (in_line[0])  
    {  
    ....  
    case 'a':  
        ....  
        {  
        ....  
        boost::tie(e1, in1) = add_edge(verts[tail], verts[head], g);  
        boost::tie(e2, in2) = add_edge(verts[head], verts[tail], g);  
        ....  
        capacity[e1] = cap;  
        capacity[e2] = 0; // row 289  
        reverse_edge[e1] = e2;  
        reverse_edge[e2] = e1;  
        }  
        ++no_alines;  
        break;  
        }  
    ....  
}  
```  
The reverse edges with capacity equal to 0 come from row 289.  
  
2. What is the right "interpretation" of the flow from 7 -> 6 being -4 in the example above?  
  
As the above discussion for the first question, this is a reverse edge, and the value, -4, is invalid.
