I need to get member name of the sequence. The code bellow works for BOOST_FUSION_DEFINE_STRUCT but doesn't for BOOST_FUSION_DEFINE_STRUCT_INLINE. I get compiler error:  
``error: incomplete type ‘boost::fusion::extension::struct_member_name<Employee, 0>’ used in nested name specifier``.  
  
```cpp  
#include <boost/fusion/adapted.hpp>  
#include <boost/fusion/mpl.hpp>  
#include <iostream>  
``  
// it works  
BOOST_FUSION_DEFINE_STRUCT(, Employee, (std::string, name)(int, age))  
  
// compile error  
//BOOST_FUSION_DEFINE_STRUCT_INLINE(Employee, (std::string, name)(int, age))  
  
int main() {  
  auto size = boost::fusion::extension::struct_size<Employee>::value;  
  std::cout << size << std::endl;  
  
  auto name = boost::fusion::extension::struct_member_name<Employee, 0>::call();  
  std::cout << name << std::endl;  
}  
```  
  
It says in doc ``BOOST_FUSION_DEFINE_STRUCT_INLINE is a macro that can be used to generate all the necessary boilerplate to define and adapt an arbitrary struct as a model of Random Access Sequence. Unlike BOOST_FUSION_DEFINE_STRUCT, it can be used at class or namespace scope.``. It sounds that both of them define sequence with the same functionality.  
Could you help?

Indeed you are right the extension boilerplate template specialization are not generated by the BOOST_FUSION_DEFINE_STRUCT_INLINE macro.  
  
It looks like something has been forgotten there.  
  
Please note though that BOOST_FUSION_DEFINE_STRUCT_INLINE is meant to be used inside an enclosing structure, like :  
  
```cpp  
struct banana {  
 BOOST_FUSION_DEFINE_STRUCT_INLINE(Employee, (std::string, name)(int, age))  
};  
```  
  
Which yields a struct which is a fusion sequence : banana::Employee.  
  
I'm looking in what can be done to add the extensions there too, but the code is quite different, in the meantime I can advise you to use instead :   
  
```cpp  
struct banana {  
 struct Employee {   
   std::string name;  
    int age;  
  };  
};  
  
BOOST_FUSION_ADAPT_STRUCT(banana::Employee, name, age)  
```
