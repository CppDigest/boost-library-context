The example for `boost::numpy::from_data` referenced in [here](https://www.boost.org/doc/libs/1_65_1/libs/python/doc/html/numpy/reference/ndarray.html#id6) is just wrong. Specifically this part:  
  
`int data[] = {1,2,3,4};`  
`np::dtype dt = np::dtype::get_builtin<int>();`  
`p::tuple shape = p::make_tuple(4);`  
`p::tuple stride = p::make_tuple(4);`  
`p::object own;`  
`np::ndarray data_ex = np::from_data(data,dt,shape,stride,own);`  
  
If you use something like this in a function to send a numpy array to python, you are going to read garbage memory. This is because `data[]` is going to be allocated on the stack and is going to get deleted when it goes out of scope. As a new user, I am under the impression that `own` would handle data’s memory management for me using some magic. You should use something like this:  
  
`std::vector<int> data{1, 2, 3, 4};`  
`np::ndarray data_ex = np::empty(shape, dt);`  
`std::copy(data.begin(), data.end(), reinterpret_cast<int*>(data_ex.get_data()));`  
  
I don’t even know why you have this as an example. It is confusing to a new user.
