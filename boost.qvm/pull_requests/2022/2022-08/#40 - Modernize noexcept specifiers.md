Unfortunately I can not accept this change, as it introduces a dependency on another Boost library. Currently, Boost QVM is decoupled from any other library, including Boost libraries. This is done for portability.  
  
It is a good idea to modernize the noexcept specifiers. One way to do it is to copy the relevant code from Boost Config and place it in boost/qvm/config.hpp, renaming the macro to BOOST_QVM_NOEXCEPT_OR_NOTHROW. I don't think this is a trivial task though.

Actually, why is this change necessary? The following program:  
  
```  
#include <exception>  
  
struct ex : std::exception  
{  
    char const * what() const throw() { return "ex"; }  
    ~ex() throw() { }  
};  
  
int main()  
{  
    throw ex();  
}  
```  
  
Works on gcc, clang and msvc without errors or issues. What compiler/version/command line requires that the `throw()` be changed to `noexcept`?

## Comment 7

> Username: georgthegreat  
> Created_at: 2022-08-23 14:26:50 UTC  
> Url: https://github.com/boostorg/qvm/pull/40#issuecomment-1224154443  

`throw()` was replaced with `noexcept` in C++11 and was officially removed from language standard [in C++17](https://en.cppreference.com/w/cpp/language/except_spec).  
  
While I have no real example of a toolchain which follows the standand literally, it is now up to toolchain developers to break the code at any time.
