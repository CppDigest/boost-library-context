# #182 - 1.75 regression: create_directories fails with "File exists" on old 2.6.32 Linux kernel [Closed]

> Username: luke-jr  
> Created at: 2021-03-29 16:21:20 UTC  
> Updated at: 2021-03-29 16:36:06 UTC  
> Closed at: 2021-03-29 16:36:06 UTC  
> Url: https://github.com/boostorg/filesystem/issues/182  

Beginning with 1.75.0, boost::filesystem::create_directories fails with "File exists", despite that it's supposed to silently accept preexisting directories. Versions up to and including 1.74.0 worked fine.  
  
```  
EXCEPTION: N5boost10filesystem16filesystem_errorE         
boost::filesystem::create_directory: File exists: "/home/user/.bitcoin"         
```  
  
strace shows:  
```  
statx(AT_FDCWD, "/home/user/.bitcoin", AT_STATX_SYNC_AS_STAT|AT_NO_AUTOMOUNT, STATX_TYPE|STATX_MODE, 0xbfe0b9dc) = -1 ENOSYS (Function not implemented)  
statx(AT_FDCWD, "/home/user", AT_STATX_SYNC_AS_STAT|AT_NO_AUTOMOUNT, STATX_TYPE|STATX_MODE, 0xbfe0b9dc) = -1 ENOSYS (Function not implemented)  
mkdir("/home/user/.bitcoin", 0777) = -1 EEXIST (File exists)  
statx(AT_FDCWD, "/home/user/.bitcoin", AT_STATX_SYNC_AS_STAT|AT_NO_AUTOMOUNT, STATX_TYPE|STATX_MODE, 0xbfe0b87c) = -1 ENOSYS (Function not implemented)  
```  
  
For reference, this system only supports a 2.6.32-based kernel.
