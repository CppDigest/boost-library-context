When I try to compile this code:  
  
```c++  
#include <iostream>  
#include <string>  
  
#include <boost/parser/parser.hpp>  
  
namespace bp = ::boost::parser;  
  
bp::rule< struct outer > outer = "outer";  
bp::rule< struct inner, std::string > inner = "inner";  
  
auto const inner_def = bp::string("X");  
auto const outer_def = bp::int_ >> inner;  
  
BOOST_PARSER_DEFINE_RULES( inner, outer );  
  
int main(int argc, const char * argv[])  
{  
	std::string toParse( "11 X" );  
	  
	auto const result = bp::parse( toParse, outer, bp::ws );  
	  
	if (result)  
	{  
		std::cout << "It parsed \n";  
	}  
	return 0;  
}  
```  
  
I get a bunch of compile errors, starting with this:  
  
> static assertion failed due to requirement 'integral_constant<long long, 1>::value < integral_constant<long long, 1>::value': Looks like you're trying to write some attribute into an out-of-bounds position in a tuple/struct.  In other words, the attribute you're parsing into does not match the default attribute used by this parser.  This may be because you passed an out-param to parse() at the top level that is not compatible with the attribute type generated by the parser you passed to parse().  
  
Well, I didn't pass an out-param to `parse()`, so I thought maybe the problem is with the explicit attribute type `std::string` that I put in the `inner` rule.  However, if I use that rule directly in the `parse()` function, i.e., `bp::parse( toParse, inner, bp::ws )`, then the code compiles.  So, what am I doing wrong?

## Comment 1

> Username: andreasbuhr  
> Created at: 2025-08-07 11:21:12 UTC  
> Url: https://github.com/boostorg/parser/issues/264#issuecomment-3163655939  

Your "outer" rule and "outer_def" definition are incompatible.  
In the definition of "outer", you specify (by omitting it) that it has no attribute.  
Then  you define outer_def so it has a std::tuple<int, std::string> attribute.  
  
You have to make this consistent.
