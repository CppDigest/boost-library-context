This looks like an excellent place to write a generic algorithm to find the last dereferenceable in a range.  It can be easily optimized to O(1) if the iterator supports bidirectional (which it doesn't have to but it might):  
  
```  
namespace detail  
{  
  
template <class FwdIt>  
FwdIt  
find_last(FwdIt first, FwdIt last, std::forward_iterator_tag)  
{  
    for(;;)  
    {  
        auto next = std::next(first);  
        if (next == last)  
            break;  
        first = next;  
    }  
    return first;  
}  
  
template <class BidIt>  
BidIt  
find_last(BidIt, BidIt last, std::bidirectional_iterator_tag)  
{  
    return std::prev(last);  
}  
  
}  // detail  
  
template <class FwdIt>  
FwdIt  
find_last(FwdIt first, FwdIt last)  
{  
    assert(first != last);  
    return detail::find_last(first, last,  
                             typename std::iterator_traits<FwdIt>::iterator_category{});  
}  
```  
  
Oh, for grins just realized this is a classic example of where we will purposefully slice an object and don't want a warning. :-)  If we pass in a `random_access_iterator_tag` it will slice when binding to `bidirectional_iterator_tag`.
