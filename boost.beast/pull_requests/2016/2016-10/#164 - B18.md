Will this line warn for certain types of x (say any unsigned bigger than size_t) ? (I might be misunderstanding the intended use...)

> Username: vinniefalco  
> Created_at: 2016-10-31 18:15:48 UTC  
> Updated_at: 2016-11-04 12:02:56 UTC  
> Url: https://github.com/boostorg/beast/pull/164#discussion_r85799765  

There are places in the WebSocket protocol which specify `uint64_t`. For example, the size of a frame can be 2^64-1 bytes. However, internally we use `std::size_t` to represent sizes which we might need to allocate. Clearly, an executable compiled for a 32-bit address model cannot allocate more than 2^32-1. This function exists to clamp a value that will be used for some type of computation involving a buffer. On 32 bit platforms it converts `std::uint64_t` to `std::uint32_t` by clamping. On 64 bit platforms it ends up doing nothing.
