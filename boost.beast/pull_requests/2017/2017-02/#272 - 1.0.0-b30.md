I'm lost. I see that `wr_block_` is cleared in the `upcall`. But when I look on line 132 of this commit, I see that if we have a `wr_block_`, then we set the state to `2`. The next time into this state machine I would expect this assert to fail. What I'm missing is how is `wr_block_` cleared between state `0` and state `2`?

> Username: vinniefalco  
> Created_at: 2017-02-27 14:53:16 UTC  
> Updated_at: 2017-02-27 16:55:24 UTC  
> Url: https://github.com/boostorg/beast/pull/272#discussion_r103222984  

`upcall` is a final state, once we perform the upcall (the call to the final completion handler) the composed operation is done and gets destroyed. When `wr_block_` is set and we suspend, it is the other blocking operation which clears `wr_block_` before the upcall (or continuing the composed operation such as in the case of `write_op`).
