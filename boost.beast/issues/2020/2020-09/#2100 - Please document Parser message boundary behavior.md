This is a documentation request.  
  
http::basic_parser apparently is designed to stop on each message boundary or body ( part ).  Therefore a person should check that the parser is actually done and not just done with the current message part, and possibly start parsing again.  
  
This may not be intuitive for some ( I didn't get it until pointed out ).  
  
It may help others in the future if this parser behavior was included in the documentation.
