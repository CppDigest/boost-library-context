@viduranga   
  
One thing that this code snippet does not do is this:  
  
```  
        // Set SNI Hostname (many hosts need this to handshake successfully)  
        if(! SSL_set_tlsext_host_name(stream.native_handle(), host))  
        {  
            beast::error_code ec{static_cast<int>(::ERR_get_error()), net::error::get_ssl_category()};  
            throw beast::system_error{ec};  
        }  
```  
  
If the web server in your docker container is sitting behind a proxy (e.g. ngnix reverse proxy) then the request may well be rejected as the proxy won't know which upstream web server to hand the request to (even if there's only one).  
  
In the above snippet, `stream` is the ssl_stream component of your protocol stack. So for your websocket it would be:  
  
`SSL_set_tlsext_host_name(ws.next_layer().native_handle(), host)`
