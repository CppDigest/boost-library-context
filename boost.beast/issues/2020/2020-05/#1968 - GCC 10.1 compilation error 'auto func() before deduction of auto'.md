Hi there,  
  
I hit a compilation error in beast related code while I'm migrating our code base to GCC 10.  
We are using boost::beast from boost 1.73.  
I isolated the issue to the following code  
```  
#include <boost/beast/http/string_body.hpp>  
#include <boost/beast/http/message.hpp>                                                                                          
#include <boost/beast/http/write.hpp>  
  
#include <boost/asio/ip/tcp.hpp>  
  
namespace net = boost::asio;  
namespace http = boost::beast::http;  
  
int main()  
{  
    net::io_context ioc;  
    net::ip::tcp::socket sk(ioc);  
  
    http::request<http::string_body> req;  
  
    http::async_write(sk, req, [](boost::system::error_code, size_t){});  
  
    return 0;  
}  
```  
[Godbold link](https://godbolt.org/z/ccuVLa)  
It gives the following error  
```  
<source>:17:71: error: use of 'auto boost::beast::http::async_write(AsyncWriteStream&, const boost::beast::http::message<isRequest, Body, Fields>&, WriteHandler&&, typename std::enable_if<(! boost::beast::http::is_mutable_body_writer<Body>::value)>::type*) [with AsyncWriteStream = boost::asio::basic_stream_socket<boost::asio::ip::tcp>; bool isRequest = true; Body = boost::beast::http::basic_string_body<char>; Fields = boost::beast::http::basic_fields<std::allocator<char> >; WriteHandler = main()::<lambda(boost::system::error_code, size_t)>; typename std::enable_if<(! boost::beast::http::is_mutable_body_writer<Body>::value)>::type = void]' before deduction of 'auto'  
  
   17 |     http::async_write(sk, req, [](boost::system::error_code, size_t){});  
  
      |                                                                       ^  
```  
The same example compiles fine with GCC 9.3.  
  
It'll be great if you can suggest some workaround for it even if it includes changes in the beast code.  
This will help us to continue with the code migration.  
  
Thanks,  
Pavel.

It turns out that, for GCC 10, this declaration from beast/http/write.hpp  
```  
template<                                                                            
    class AsyncWriteStream,                                                       
    bool isRequest, class Body, class Fields,                                     
    BOOST_BEAST_ASYNC_TPARAM2 WriteHandler =                                         
        net::default_completion_token_t<                                          
            executor_type<AsyncWriteStream>>>                                     
BOOST_BEAST_ASYNC_RESULT2(WriteHandler)                                           
async_write(                                                                         
    AsyncWriteStream& stream,                                                        
    message<isRequest, Body, Fields> const& msg,                                  
    WriteHandler&& handler =                                                      
        net::default_completion_token_t<                                          
            executor_type<AsyncWriteStream>>{}                                    
#ifndef BOOST_BEAST_DOXYGEN                                                       
    , typename std::enable_if<                                                    
        ! is_mutable_body_writer<Body>::value>::type* = 0                            
#endif                                                                               
    );  
```  
is not the same as the definition from beast/http/impl/write.hpp  
```  
template<                                                                            
    class AsyncWriteStream,                                                          
    bool isRequest, class Body, class Fields,                                        
    class WriteHandler>                                                              
BOOST_BEAST_ASYNC_RESULT2(WriteHandler)                                              
async_write(                                                                         
    AsyncWriteStream& stream,                                                        
    message<isRequest, Body, Fields> const& msg,                                     
    WriteHandler&& handler,                                                          
    typename std::enable_if<                                                         
        ! is_mutable_body_writer<Body>::value>::type*)  
```  
and the compiler gives the error because it can't see the definition of the function to deduce the auto.  
The error disappeared when I changed this  
```  
BOOST_BEAST_ASYNC_TPARAM2 WriteHandler =                                         
        net::default_completion_token_t<                                          
            executor_type<AsyncWriteStream>>  
```  
to this  
```  
class WriteHandler  
```
