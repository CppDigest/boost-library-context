I am trying to code a C++ server game using Boost.Asio. I also want to support websocket sessions so that I am trying to use Boost.Beast. Then I downloaded the `boost_1_73_0` sources from the boost website and I've built it. It worked fine when I just used Boost.Asio but as soon as I make an include of `<boost/beast/core.hpp>` (I still don't even use the beast stuff) I got errors. I asked  support on some discords and it seems it could either be a problem coming from beast compatibility or a configuration issue (my cmake would be wrongly configured).  
  
### Version of Beast  
  
Looking at the `<boost/beast/version.hpp>` I guess this is 290 (`#define BOOST_BEAST_VERSION 290`). I'm using boost 1.73.  
  
### Steps necessary to reproduce the problem  
  
Start a cmake project that includes boost (`find_package`). Make a class and include `<boost/beast/core.hpp>` using boost 1.73.0. I use GCC 10.1.0.
