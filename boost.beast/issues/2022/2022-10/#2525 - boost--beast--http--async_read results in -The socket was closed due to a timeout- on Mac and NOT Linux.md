### Steps necessary to reproduce the problem  
  
I have a modified version of the async service found at [http_server_async_ssl](https://www.boost.org/doc/libs/master/libs/beast/example/http/server/async-ssl/http_server_async_ssl.cpp) that I am using in a project that I host on [github](https://github.com/mehoggan/dag_scheduler). The project is large enough that breaking the code out might not be worth the effort as I can provide the details here.  
  
Please note that the code works as expected in Fedora 36 but on Mac it fails with an error code that contains a message:  
  
"The socket was closed due to a timeout"  
  
The starting of the server starts at: https://github.com/mehoggan/dag_scheduler/blob/2b41e3d871d421597a2b14cea2eada2acb072a65/bin/main.cpp#L47  
  
Within the constructor for the WorkflowService the starting of the HTTPS async server starts here:  
https://github.com/mehoggan/dag_scheduler/blob/2b41e3d871d421597a2b14cea2eada2acb072a65/lib/workflow_service.cxx#L95  
  
At which point it awaits for a connection. I use Postman or the Web Browser to make a GET request to `https://localhost:8888/index.html`.  
  
Once the connection is accepted at:  
https://github.com/mehoggan/dag_scheduler/blob/2b41e3d871d421597a2b14cea2eada2acb072a65/lib/workflow_service.cxx#L158  
  
A HTTPSSession is created. in the HTTPSSession::run method an executor is obtained, the SSL handshake is performed and subsequently the session attempts to read in the request. This may all be traced from:  
https://github.com/mehoggan/dag_scheduler/blob/2b41e3d871d421597a2b14cea2eada2acb072a65/lib/https_session.cxx#L64  
  
I have a 30 second timeout set on the read here:  
https://github.com/mehoggan/dag_scheduler/blob/2b41e3d871d421597a2b14cea2eada2acb072a65/lib/https_session.cxx#L99  
  
On my Mac the timeout occurs, but on my Fedora 36 box the timeout does not occur. I have tried to increase the timeout but that did nothing besides make me wait longer for it to time out. One thing that I think might be resulting in the issue is that the project builds OpenSSL from source to mitigate environmental differences between Mac and Linux. Any help would be greatly appreciated. I have tried to step through the code but nothing is glaring out at me as the cause for the timeout. My assumption is that OpenSSL is not reading in from the socket on my Mac.
