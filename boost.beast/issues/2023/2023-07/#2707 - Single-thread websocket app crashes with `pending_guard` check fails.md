# #2707 - Single-thread websocket app crashes with `pending_guard` check fails [Closed]

> Username: zptan

> Updated at: 2023-08-03 04:19:32 UTC

> Url: https://github.com/boostorg/beast/issues/2707  

My project is a client that subscribes (websocket) data from a server, the server pushes data to the client when there is update, and the client will ping the server to keep the connection alive  
  
When testing I got crash by running DEBUG version, but RELEASE version runs well  
```  
my_proj: /usr/include/boost/beast/core/detail/stream_base.hpp:80: boost::beast::detail::stream_base::pending_guard::pending_guard(bool &): Assertion `! b_' failed.  
[1]    3905731 IOT instruction (core dumped)  
```  
  
### Version  
  
Beast 300  
Boost 1.74  
  
### Steps necessary to reproduce the problem  
It is a private project so can not disclose, just show how it works  
  
- `async_write()` only called when sending request to subscribe data  
- After subscribe successfully only `async_read()` and `async_ping()` will be called  
- No multi-threading  
- When connection ends, the caller will re-create a MyClient instance to subscribe data  
  
```cc  
class MyClient {  
    unsigned ping_interval_;  
    uint64_t last_ping_ts_;  
  
    using read_buffer_t = boost::beast::flat_buffer;  
    read_buffer_t read_buffer_;  
    read_buffer_t response_buffer_;  
  
    std::mutex loop_mtx_;  
public:  
    ...  
  
    void Loop(const atomic_bool* shutdown) {  
        std::unique_lock loop_lock{loop_mtx_, std::try_to_lock};  
        if (!loop_lock.owns_lock()) {  
            return;  
        }  
  
        while (!*shutdown) {  
            ws_ = make_unique<...>(io_context_, ssl_context_);  
            StartStream(); // Resolve, Connect, Handshake, etc  
            io_context_.restart();  
  
            while (!io_context_.stopped() && !*shutdown) {  
                boost_error_code_t ec;  
                io_context_.run_one(ec);  
                if (ec) {  
                    break;  
                }  
            }  
  
            io_context_.stop();  
              
            if (!reconnect_ || *shutdown) {  
                break;  
            }  
            std::this_thread::sleep_for(5s);  
        }  
    }  
  
    void OnHandshake(const boost_error_code_t handshake_ec) {  
        return OnStreamStart();  
    }  
  
    void OnStreamStart() {  
        ReadStream(  
            &response_buffer_,  
            [this](const boost_error_code_t connection_error, const std::string_view data) {  
                OnConnectionResponseRead(connection_error, data);  
        });  
    }  
  
    void OnConnectionResponseRead(const boost_error_code_t ec, const std::string_view raw_data) {  
        Send(request_str_, [this](const boost_error_code_t connection_error, const std::size_t write_size) {  
            OnSubscriptionRequestSend(connection_error, write_size);  
        });  
    }  
  
    void Send(std::string data, auto&& on_write) {  
        auto        data_storage = std::make_unique<std::string>(std::move(data));  
        const auto& data_to_send = *data_storage;  
        ws_->async_write(  
            boost::asio::buffer(data_to_send),  
            [data_storage = std::move(data_storage),  
             on_write     = std::move(on_write)](const auto ec, const auto bytes_transferred) {  
                on_write(ec, bytes_transferred);  
            });  
    }  
  
    void OnSubscriptionRequestSend(const boost_error_code_t ec, const std::size_t bytes_written) {  
        ReadStream(  
            &response_buffer_,  
            [this](const boost_error_code_t connection_error, const std::string_view data) {  
                OnSubscriptionResponseRead(connection_error, data);  
            });  
    }  
  
    void OnSubscriptionResponseRead(const boost_error_code_t ec, const std::string_view data) {  
        if (IsSubscriptionOK(data)) {  
            ReadStream(&read_buffer_, [this](const boost_error_code_t ec, const std::string_view raw_data) {  
                StreamReadStep(ec, raw_data);  
            });  
            return;  
        }  
  
        // LOG error and retry  
    }  
  
    void ReadStream(read_buffer_t* read_buffer, auto&& on_read) {  
        read_buffer->clear();  
        ws_->async_read(  
            *read_buffer,  
            [on_read = std::move(on_read), read_buffer](const auto ec, const auto /* bytes_transferred */) {  
                on_read(ec, std::string_view(static_cast<const char*>(read_buffer->cdata().data()), read_buffer->size()));  
        });  
    }  
  
    void StreamReadStep(const boost_error_code_t ec, const std::string_view raw_data) {  
        OnRead(raw_data);  
        ReadStream(  
            &read_buffer_,  
            [this](const boost_error_code_t err_code, const std::string_view read_raw_data) {  
                StreamReadStep(err_code, read_raw_data);  
        });  
    }  
  
    void OnRead(const std::string_view data) {  
        ParseData(data);  
          
        // Only ping after data processed  
        if (GetCurrentTimestamp() - last_ping_ts_ >= ping_interval_) {  
            Ping();  
        }  
    }  
  
    void Ping() {  
        ...  
        async_ping(ping_data, [](...){});  
        last_ping_ts_ = GetCurrentTimestamp();  
    }  
};  
```  
  
### All relevant compiler information  
LLVM 14.0  
  
### More info  
I found the [doc](https://www.boost.org/doc/libs/1_74_0/boost/beast/core/detail/stream_base.hpp)  
```cc  
        explicit  
        pending_guard(bool& b)  
            : b_(b)  
        {  
            // If this assert goes off, it means you are attempting  
            // to issue two of the same asynchronous I/O operation  
            // at the same time, without waiting for the first one  
            // to complete. For example, attempting two simultaneous  
            // calls to async_read_some. Only one pending call of  
            // each I/O type (read and write) is permitted.  
            //  
            BOOST_ASSERT(! b_);  
            b_ = true;  
        }  
```  
  
So it seems that there are concurent `async_read()`s or `async_write()`s, but I can not find them from the code.  
  
So RELEASE build removes `BOOST_ASSERT(...)`, if concurent `async_read()`s or `async_write()`s cause bug, why only DEBUG build enable the check?
