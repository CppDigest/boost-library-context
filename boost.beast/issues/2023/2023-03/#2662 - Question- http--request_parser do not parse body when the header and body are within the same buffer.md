### Version of Beast  
BOOST_BEAST_VERSION=266  
  
### Steps necessary to reproduce the problem  
When I am using http::request_parser to parse a reuqest generated by curl commands:  
`curl http://127.0.0.1:8080/112 -v -d "1212312312312"` and it seems that `http::reuqest_parser` do not parse the HTTP body when the header and body are within the same buffer.  
  
Here is an example in function case1:  
```cpp  
#include <boost/beast/http/parser.hpp>  
#include <boost/beast/http/string_body.hpp>  
#include <boost/optional/optional_io.hpp>  
#include <iostream>  
#include <string>  
  
using namespace std;  
using namespace boost::beast;  
  
void case1() {  
  std::string header_and_body =  
      "POST /112 HTTP/1.1\r\nHost: 127.0.0.1:8080\r\nUser-Agent: "  
      "curl/7.68.0\r\nAccept: */*\r\nContent-Length: 13\r\nContent-Type: "  
      "application/x-www-form-urlencoded\r\n\r\n1212312312312";  
  
  http::request_parser<http::string_body> parser;  
  boost::beast::error_code ec;  
  
  // trying to parse header and body together  
  parser.put(  
      boost::asio::buffer(header_and_body.data(), header_and_body.size()), ec);  
  if (ec) {  
    cout << ec.message() << endl;  
  }  
  
  parser.put_eof(ec);  
  if (ec) {  
    // output: partial message  
    cout << ec.message() << endl;  
  }  
  
  // output: 13 (the length of body)  
  cout << parser.content_length_remaining() << endl;  
  
  // output: 0  
  cout << parser.is_done() << endl;  
}  
  
void case2() {  
  std::string header =  
      "POST /112 HTTP/1.1\r\nHost: 127.0.0.1:8080\r\nUser-Agent: "  
      "curl/7.68.0\r\nAccept: */*\r\nContent-Length: 13\r\nContent-Type: "  
      "application/x-www-form-urlencoded\r\n\r\n";  
  std::string body = "1212312312312";  
  
  http::request_parser<http::string_body> parser;  
  boost::beast::error_code ec;  
  
  // parse header  
  parser.put(boost::asio::buffer(header.data(), header.size()), ec);  
  if (ec) {  
    cout << ec.message() << endl;  
  }  
  
  // parse body  
  parser.put(boost::asio::buffer(body.data(), body.size()), ec);  
  if (ec) {  
    cout << ec.message() << endl;  
  }  
  
  parser.put_eof(ec);  
  if (ec) {  
    cout << ec.message() << endl;  
  }  
  
  // output: 0  
  cout << parser.content_length_remaining() << endl;  
  
  // output: 1  
  cout << parser.is_done() << endl;  
}  
  
int main() {  
  case1();  
  case2();  
}  
```  
  
In case1, I tried to parse a buffer read from socket that contains http header and body, but it seems that the parser only parsed the header and then `parser.content_length_remaining()` is set to 13, which is the Content-Length of the body.  
In case2, I tried to separate the header and body, and the parser worked correctly.  
  
Is there any way to parse http header and body within a buffer? Cause a buffer often contains header and body together when we are receiving a http request from socket.  
  
### All relevant compiler information  
g++-10 (Ubuntu 10.3.0-1ubuntu1~20.04) 10.3.0
