Beast Version: 70  
Compiler: MSVC v141, compiled into an x86 binary  
  
Calling async_write returns the error `Websocket connection failed due to a protocol violation.`   
  
```  
auto rtn = handle_read();  
  
timer_.expires_from_now(std::chrono::seconds(15));  
  
// Write Auth Message  
ws_.async_write(boost::asio::buffer(&rtn[0], rtn.size()),  
    strand_.wrap(std::bind(&connection::on_write,  
        shared_from_this(), std::placeholders::_1)));  
```  
  
handle_read() is a function that returns a std::string based on the message read from the server. Sometimes a uint64_t that's been converted to an std::string, sometimes a hash. I've validated with a debugger that the string returned from handle_read is valid, printing and changing the string works fine as well as just turning it into a boost::asio::mutable_buffer, only when writing it does this occur.

https://github.com/vinniefalco/Beast/blob/fa1ac16584f03590ecea617a0dc2d8c86e48cda5/include/beast/websocket/impl/write.ipp#L485
