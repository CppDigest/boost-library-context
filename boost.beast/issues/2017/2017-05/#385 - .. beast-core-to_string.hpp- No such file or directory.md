Were you calling `to_string` in your code, and then you updated to a newer Beast and got the compile error? The function `to_string` has been removed. If you want to convert buffers to a string, use `boost::lexical_cast` and the `buffers` function. Like this:  
  
```  
    boost::lexical_cast<std::string>(beast::buffers(b));  
```  
  
Where **b** is your **ConstBufferSequence**  
  
The WebSocket example illustrates this usage:  
https://github.com/vinniefalco/Beast/blob/dab679cb0a3cf51a5ecf4e52a832edcd88ff5609/examples/websocket_example.cpp#L34  
  
Hope this helps!
