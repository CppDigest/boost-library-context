The simple answer is to make the test suite work.  
  
The test suite used internally is a tad leaky in that it relies on these macros being defined. Previously, this was always true because the macros were used internally by the implementation and they leaked out.  
  
In this case, we simply define a dummy `fwd.hpp` header file so that the normal test utilities we already use like `UNORDERED_TEST` compile and run with minimal changes required.  
  
There's a bit of messy history here. This `fwd.hpp` header has always existed and its macros always relied upon by the test runner. There were more macros in implementation.hpp that were also required. I solved this for foa by just including `implementation.hpp` directly as it defined a lot of macros used by the tests that existed for fca.  
  
But for this new concurrent container, I wanted to avoid relying on the existing implementation.hpp so this was the bare minimum that needed to be relocated from implementation.hpp to make the tests work.  
  
I'm happy you brought this up because I realized that the foa fwd headers were erroneously including this fwd.hpp so I've now since relocated those includes.
