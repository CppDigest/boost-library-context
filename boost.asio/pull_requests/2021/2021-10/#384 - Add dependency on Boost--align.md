There are some automated tests for Boost as a whole (whether it builds and installs with CMake), but they can't catch this. There are no outside tests that test every library one by one; such tests need to be enabled in the library repos.  
  
Example workflows are [this one](https://github.com/boostorg/timer/blob/9ee4c9cd9dec4f4ab5c38ffb9cb9978427916ee0/.github/workflows/ci.yml#L200) that tests use with `add_subdirectory` and [this one](https://github.com/boostorg/timer/blob/9ee4c9cd9dec4f4ab5c38ffb9cb9978427916ee0/.github/workflows/ci.yml#L255) that tests use after installation.
