# #89 - 1.66 API breaking change in io_context constructor [Closed]

Hello,  
  
1.66 introduced an API breaking change in the constructor of `io_context`.  
  
`io_context` takes an `int` while `io_service` took a `std::size_t`. Compare [v1.66](http://www.boost.org/doc/libs/1_66_0/doc/html/boost_asio/reference/io_context/io_context.html) and [v1.65](http://www.boost.org/doc/libs/1_65_0/doc/html/boost_asio/reference/io_service/io_service.html). This change breaks all code that stored a `std::size_t` which later got passed to the `io_service` constructor due to a now narrowing conversion.  
  
I think this should either be fixed or noted on http://www.boost.org/doc/libs/1_66_0/doc/html/boost_asio/net_ts.html as `io_service` is not a strict typedef but a different type.  
  
For example, see https://github.com/Microsoft/cpprestsdk/issues/654.  
  
thanks,  
Stephan
