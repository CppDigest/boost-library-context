Since Boost.MSM provides best-in-class performance, and, arguably, functionality, I would like to use it in a production code where the final execution speed/latency is a paramount, and not the ease of use/adoption, or executable size, or compilation time. In my use case I'll have to make the very base of the infrastructural/chassis code to depend on MSM. However, the activity of this project, or should I say lack of it, is a bit worrying.  
  
So I kindly ask the maintainers to share their vision on the MSM future.  
  
Thanks!
