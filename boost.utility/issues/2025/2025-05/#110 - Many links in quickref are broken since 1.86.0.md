1.85.0: https://www.boost.org/doc/libs/1_85_0/libs/utility/doc/html/utility/ref.html  
1.86.0: https://www.boost.org/doc/libs/1_86_0/libs/utility/doc/html/utility/ref.html  
  
Many links (like almost every one in the top row) lead to `../`. The cause is this BoostBook commit: https://github.com/boostorg/boostbook/commit/b1fef2e5b9c158b91053e20d3303706a52f5bb00.  
  
This error motivates me to ask you to consider switching from BoostBook's Doxygen converter to [Docca](https://github.com/boostorg/docca) (specifically, the Python version).  
  
Here is how I use it in JSON: https://github.com/boostorg/json/blob/develop/doc/Jamfile#L19-L43.  
Here's the result: https://www.boost.org/doc/libs/develop/libs/json/doc/html/json/ref.html.  
Here's a WIP result using a very different custom format: https://1080.json.prtest2.cppalliance.org/libs/json/doc/html/ref.html (look into docs for overloaded functions like `parse`).  
  
The main difference in approach is that I separated the process of data collection from Doxygen output and the process of producing output.
