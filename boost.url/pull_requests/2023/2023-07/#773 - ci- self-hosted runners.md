## Comment 10

> Username: sdarwin  
> Created_at: 2023-07-22 01:08:42 UTC  
> Url: https://github.com/boostorg/url/pull/773#issuecomment-1646366359  

1. Upgraded CMake. Job succeeded.  
  
2. Taking a long time to get started.  
  
Each job gets its own instance, which is launched, boots up, connects to the system, GHA detects the runner, sends over the job. After one job, the machine is destroyed. The process happens mostly in parallel.  Windows machines are especially slow to start up, compare to Linux. Their boot time adds a noticable delay.    
  
Next, the AWS instances are sized as "large", which is the smallest feasible size. The machines are not xlarge or 2xlarge, etc. GitHub is provisioning more powerful instances, or making more CPU available to each job. We could throw more money at the issue. Or just live with the current times.  GitHub runners might have double the CPU.   
  
The system should eventually prefer and switch to GitHub runners, and only use self-hosted runners at the end of the month when necessary.
